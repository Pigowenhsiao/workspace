# CLAUDE.md

1) Build 與 Run 約定
- 目的：確保可預期的執行結果與可回滾性。
- 流程順序：Build → Test → Verify → Release
- 輸出：明確的狀態報告（PASS/FAIL），以及回滾指令。

2) Architecture Boundaries（分層邊界）
- CLAUDE.md：專案契約、核心約束、不可變原則。
- rules：路徑、語言、檔案型別等規則集合，按需載入。
- skills：可執行的工作流與知識模組，按需加載。
- hooks：硬性審核與自動化檢查點，阻止不合規操作。
- subagents：隔離執行的獨立實例，處理長時間任務或高輸出任務。
- verifier：驗證閉環機制，定義成功/失敗的標準與回滾。
- memory/designated store：長期記憶與證據鏈，避免喪失上下文。

3) NEVER 清單（禁止做什麼）
- 未經驗證的外部操作直接落地（如直接修改核心資料庫、直接對外部署未驗證版本）。
- 以單次回合替代完整驗證的任務完成判定。
- 超出允許工具集的高權限操作（未經 hooks/policy 允許）。

4) ALWAYS 清單（必須執行的操作）
- 每次任務前：定義驗證標準、輸入/輸出介面、預期結果。
- 設定 Plan Mode：探索/計畫先行，執行再跑驗證。
- 使用 Subagents 進行長時間/高輸出的任務分離。
- 設置 Hooks：對敏感路徑、關鍵檔案、關鍵設定進行審核與鎖定。
- 設置 Verifier：最低層命令、單元測試、日誌對比；必要時視覺差異與合同測試。
- Prompt Caching 控制：靜態系統 prompt 與動態資訊分離，動態資訊以單獨輸入注入。

5) Plan Mode 與任務執行模式
- 觸發條件：複雜任務、跨模組變更、需要證據支持的決策。
- 流程：
  - 探索/Plan：只讀工具集；蒐集證據與路徑，產出計畫書。
  - 審核/批準：你或指定審核者審核 Plan。
  - 執行：依計畫執行，分工給 Subagents，主線只收摘要。
  - 驗證與回滾：Verifier 驗證結果，失敗時自動回滾並記錄。
- 輸出物：Plan 文檔、風險評估、下一步、回滾指令。

6) Skills 設計原則
- 輸入/輸出清晰定義、前置條件與停止條件。
- 支援 defer_loading 與 supporting files。
- 適時設定 disable-model-invocation，避免模型自行觸發敏感步驟。
- 常用技能先提供穩定的基礎能力（例如：配置檢查、煙霧測試、審核報告、變更日誌更新）。

7) Subagents 使用原則
- 目的：任務隔離與輸出降噪，主線聚焦摘要與決策。
- 要點：隔離工具集合、獨立上下文、完成後回報要點摘要。
- 適用場景：大規模代碼掃描、長時間測試、深度審核。

8) Hooks 的實作要點
- PostToolUse、Edit、Notification 等觸發點。
- 常見動作：自動執行條件檢查、格式化、通知、阻斷未授權修改。
- 原則：避免過度自動化導致難以追蹤的副作用，確保可審計。

9) Verifier 的分層
- 底層：命令退出狀態、lint、typecheck、單元測試。
- 中層：集成測試、日誌一致性、輸出契約比對。
- 高層：生產日誌、監控指標、人工審查清單。
- 任何任務若未達成驗證，需回滾且記錄風險與下一步修正。

10) Prompt Caching 策略
- 靜態內容：System Prompt、Tool Definitions、CLAUDE.md 固定不動。
- 動態內容：對話歷史、日期、實時狀態以單獨欄位注入，避免污染快取。
- 指定時機：如需動態信息，使用單獨輸入注入，而非改動靜態前綴。

11) HANDOFF 與 Compact Instructions
- HANDOFF.md：跨會話接棒的進展、證據、下一步、需要協助的部分。
- Compact Instructions：需要壓縮時保留的核心資訊（架構決策、修改摘要、驗證狀態、風險與回滾）。
- 目的：讓下一任務快速接手，降低壓縮造成的遺漏。

12) 验证與風險管理
- 每次修改前定義驗證條件與成功標準。
- 建立可觸發的回滾路徑與自動化回滾機制。
- 對重大改動建立回滾演練與風險降至最低的策略。
