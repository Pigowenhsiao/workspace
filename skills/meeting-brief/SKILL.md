# Meeting Brief Skill

## 什麼時候用

- 使用者說「幫我準備明天的會議」
- 使用者說「整理一下這個客戶的背景」
- 使用者說「生成會議簡報」
- 需要結合 vault 筆記與即時資料

## 工作流程

### Step 1: 確認會議資訊

先向使用者確認必要資訊：

```
必要：
- 會議主題/名稱
- 會議時間
- 參與人員（盡量詳細）

可選：
- 會議連結
- 議程文件
- 優先想討論的話題
```

### Step 2: 資料收集

從以下來源收集資訊：

1. **使用者提供的資料**
   - 直接貼上的文字、檔案、連結
   - 稍後 meeting 模式下的對話內容

2. **Vault 搜尋**
   - 搜尋相關客戶、專案、人物的既有筆記
   - 搜尋相關歷史會議記錄
   - 使用 `find-skills` 協助搜尋

3. **外部 Connector（如已啟用）**
   - Email connector：抓取與該人員/公司的歷史郵件
   - Calendar connector：抓取過去相關會議記錄

### Step 3: 整理與生成

整理收集到的資料，生成會前簡報：

```markdown
# [會議名稱] 會前簡報

## 基本資訊
- 時間：[時間]
- 地點/連結：[地點/連結]
- 參與人員：[人員清單]

## 會議目的
[一段話說明這次會議想要達成的目標]

## 參與人員背景
### [人員A]
- 職位：[職位]
- 背景：[從 vault 或使用者處獲得的背景]
- 過去互動：[相關歷史]

### [人員B]
...（同上格式）

## 議程
| 時間 | 主題 | 主講人 | 準備重點 |
|------|------|--------|----------|
| 10:00 | 開場 | [姓名] | [準備重點] |
| ... | ... | ... | ... |

## 重點話題
### 預期討論
[列出預期會討論的重點話題]

### 需要確認的事項
[列出需要在會議中確認的具體問題]

### 需要澄清的疑點
[列出仍不清楚、需要進一步了解的點]

## 參考資料
- [[相關筆記1]]
- [[相關筆記2]]
- [相關檔案]

## 我的觀察與建議
[根據收集到的資料，給出觀察與建議]
```

### Step 4: 交付與確認

1. 將生成的簡報展示給使用者
2. 詢問是否有需要補充或修改
3. 若有需要，根據反饋調整

### Step 5: 會後整理（可選）

會議結束後：
- 將簡報更新，加入會議結論
- 將新資訊存入 vault 相關筆記
- 標記待追蹤事項

## 邊界條件

| 情況 | 處理方式 |
|------|----------|
| 使用者未提供會議主題 | 先詢問，再繼續 |
| 沒有相關 vault 筆記 | 說明「目前沒有找到相關背景資料，請補充」 |
| 外部 connector 未啟用 | 使用手動輸入模式，不假裝有自動化 |
| 資訊過少無法生成有意義簡報 | 告知使用者需要更多資訊 |
| 會議時間已過 | 提醒並詢問是否需要調整 |

## 示例

### 典型使用場景

**輸入：**
> 幫我準備明天跟 Acme Corp 的會議，客戶是 John Smith，他們是主要供應商。

**輸出：**
```markdown
# Acme Corp 商務會議 會前簡報

## 基本資訊
- 時間：明天 14:00
- 地點：線上會議
- 參與人員：John Smith (採購主管)

## 參與人員背景
### John Smith
- 職位：Acme Corp 採購主管
- 背景：[從 vault 搜尋到的相關資訊]
- 過去互動：[歷史溝通記錄]

## 會議重點
[根據收集到的資料整理的重點]

...
```

## 限制說明

- 本 skill 目前以手動輸入資料為主
- 若需要完整自動化，需啟用 email/calendar connector
- 建議在 vault 中建立相關客戶/專案筆記，以便未來快速取用
