# Mail Triage Skill

## 什麼時候用

- 使用者說「幫我整理這些郵件」
- 使用者說「把郵件分流一下」
- 使用者說「哪些郵件需要跟進」
- 使用者說「把這個郵件存到 vault」

## 工作流程

### Step 1: 收集郵件

```
必要：
- 郵件內容（可直接粘貼或提供郵件文字）

可選：
- 寄件人
- 主題
- 時間
```

### Step 2: 分析郵件

分析每封郵件：

```markdown
## 分析維度

### 需要行動？
- 是 → 加入行動桶
- 否 → 加入參考桶

### 緊急程度
- 🔴 高：需要 24 小時內回覆
- 🟡 中：需要本週回覆
- 🟢 低：可延後

### 類型
- 請求
- 更新
- 邀請
- 通知
- 垃圾郵件
```

### Step 3: 分流決策

將郵件分配到相應桶：

```markdown
## 分流結果

### 🔴 需要行動（高優先）
| 郵件 | 行動 | 截止 |
|------|------|------|
| [主題] | [行動描述] | [時間] |

### 🟡 需要行動（一般）
| 郵件 | 行動 | 截止 |
|------|------|------|

### 🟢 可延後
| 郵件 | 原因 |
|------|------|

### 🗑️ 垃圾郵件
| 郵件 | 原因 |
|------|------|
```

### Step 4: 存入 Vault（如需要）

```markdown
# Email - [寄件人] - [日期]

## 基本資訊
- 寄件人：[姓名/郵箱]
- 主題：[主題]
- 日期：[時間]
- 緊急程度：[級別]

## 內容摘要
[郵件內容摘要]

## 需要跟進的事項
- [ ] [行動1]
- [ ] [行動2]

## 相關連結
- [相關郵箱/會議]
```

## 分流規則模板

### 規則 1：行動導向
```
如果郵件包含以下關鍵詞：
- 「請回覆」
- 「需要你」
- 「截止」
→ 加入行動桶
```

### 規則 2：分類標準
```
- 內部郵件 → 團隊桶
- 客戶郵件 → 客戶桶
- 會議邀請 → 日曆桶
- 推廣郵件 → 歸檔桶
```

## 邊界條件

| 情況 | 處理方式 |
|------|----------|
| 沒有 Connector | 告知使用者目前為手動模式 |
| 郵件過多 | 建議分批處理或建立規則 |
| 郵件包含附件 | 說明需要單獨處理附件 |
| 隱私內容 | 標記為私密，不存入 vault |

## 示例

### 典型使用場景

**輸入：**
> 幫我整理這幾封郵件：
> 1. [郵件1內容]
> 2. [郵件2內容]

**輸出：**
```markdown
## 分流結果

### 🔴 需要行動
| 郵件 | 行動 |
|------|------|
| 關於預算會議 | 週三前回覆確認人數 |

### 🟢 可延後
| 郵件 | 原因 |
|------|------|
| 產品更新通知 | 已讀，無需行動 |
```
