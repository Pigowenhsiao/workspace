# Week Planner Skill

## 什麼時候用

- 使用者說「幫我規劃這週的工作」
- 使用者說「整理一下這週要做的事」
- 使用者說「生成週計劃」
- 使用者有一堆零散任務需要系統化

## 工作流程

### Step 1: 收集任務資訊

向使用者收集：

```
必要：
- 目前有的待辦事項（可以散乱形式提供）
- 這週是否有明確的截止期限

可選：
- 優先級偏好（哪類事情優先）
- 時間限制（會議、旅遊等）
- 想要的工作節奏
```

### Step 2: Vault 搜尋（可選）

搜尋 vault 中可能相關的資訊：
- 待追蹤的項目（pending projects）
- 上週未完成的任務
- 已知的重要截止日期

### Step 3: 分析與分類

將收集到的任務分析：

```
分類維度：
1. 緊急程度
   - 緊急且重要
   - 重要但不緊急
   - 可延後
   - 可委託

2. 任務類型
   - 深度工作（需要專注的任務）
   - 例行性任務
   - 溝通協調任務
   - 學習/準備任務

3. 時間需求
   - 大任務（需要拆分的）
   - 可以在1-2小時內完成的
   - 碎片時間可做的
```

### Step 4: 生成週計畫

根據分析生成結構化週計畫：

```markdown
# 本週計畫 [日期區間]

## 本週主題
[一句话描述這週的核心目標或主題]

## 重要截止
| 事項 | 截止時間 | 優先級 |
|------|----------|--------|
| [事項1] | [時間] | 🔴 高 |
| [事項2] | [時間] | 🟡 中 |

## 週一至週五規劃

### 週一
**主題：** [這天的主题]
**重點任務：**
- [ ] [任務1] ⏱ [預估時間]
- [ ] [任務2] ⏱ [預估時間]

**可選/碎片時間：**
- [ ] [可選任務]

---

### 週二
...（同上格式）

## 本週練習事項
| 任務 | 預估時間 | 說明 |
|------|----------|------|
| 任務A | 2h | [說明] |
| 任務B | 30m | [說明] |

## 深度工作時間區塊
[標記適合深度工作的時間段]

## 預期產出
- [產出1]
- [產出2]

## 風險與對策
| 風險 | 對策 |
|------|------|
| [風險] | [對策] |

## 週檢查點
- [ ] 週三：檢視進度，調整下半週
- [ ] 週五：回顧本週，準備下週
```

### Step 5: 交付與調整

1. 展示週計畫
2. 詢問是否需要調整
3. 根據反饋修改

## 邊界條件

| 情況 | 處理方式 |
|------|----------|
| 使用者沒有提供任何任務 | 詢問「這週有哪些事情需要處理？」 |
| 任務過多無法一週完成 | 幫助優先級排序，標記可延後的任務 |
| 有衝突的截止日期 | 指出衝突並建議協商時程 |
| 使用者沒有明确優先級偏好 | 使用默認優先級（緊急 > 重要 > 可延後） |
| 沒有 vault 資料 | 以使用者提供的資訊為主，不強行搜尋 |

## 示例

### 典型使用場景

**輸入：**
> 幫我規劃這週，週三有個報告要交，週五要跟客戶開會，還有一些零碎的郵件要回。

**輸出：**
```markdown
# 本週計畫 2026-04-14 ~ 2026-04-18

## 本週主題
完成報告 + 準備客戶會議

## 重要截止
| 事項 | 截止時間 | 優先級 |
|------|----------|--------|
| 報告初稿 | 週三 17:00 | 🔴 高 |
| 客戶會議準備 | 週五 14:00 | 🔴 高 |
| 郵件整理 | 週五前 | 🟡 中 |

## 週一至週五規劃

### 週一
**主題：** 啟動報告 + 收集資料
**重點任務：**
- [ ] 確認報告方向 ⏱ 1h
- [ ] 收集相關資料 ⏱ 2h
- [ ] 回覆緊急郵件 ⏱ 30m

### 週二
**主題：** 報告撰寫
...
```

## 限制說明

- 目前以手動輸入模式為主
- 外部 calendar/email 未整合
- 需要使用者主動提供任務資訊
- Vault 筆記有助於了解任務背景
