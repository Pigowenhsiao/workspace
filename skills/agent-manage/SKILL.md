# Agent Manage Skill

## 什麼時候用

- 使用者說「查看目前有哪些 agents」
- 使用者說「幫我調整某個 agent 的職責」
- 使用者說「停用某個不用的 agent」
- 使用者說「重新命名這個 agent」
- 使用者說「規劃一下我的 agents 怎麼分工」

## 工作流程

### Step 1: 了解現有 Agents 清單

從以下來源收集 agent 清單：

```
來源：
1. Vault 中的 agents 目錄（~/Documents/Agent/）
2. Workspace 中的 skills 目錄
3. 使用者對話中提到的 agents
4. 設定文件（如 openclaw.json）中的 agent 設定
```

整理清單：

```markdown
## 目前 Agents 清單

| Agent 名稱 | 職責 | 狀態 | 最後使用 |
|-----------|------|------|----------|
| [名稱1] | [職責描述] | 啟用/停用 | [時間] |
| [名稱2] | [職責描述] | 啟用/停用 | [時間] |
```

### Step 2: 分析現況

根據清單分析：

```
分析維度：
1. 職責重疊（多個 agents 做類似的事）
2. 閒置 agents（長時間未使用）
3. 協作缺口（某類任務沒有對應 agent）
4. 優先順序（哪些是核心 agents）
```

### Step 3: 根據需求執行操作

**檢視模式：**
直接展示清單和分析，不做變更。

**調整模式：**
1. 了解要調整的內容
2. 確認調整範圍（影響哪些檔案）
3. 執行調整
4. 驗證調整結果

**停用模式：**
1. 確認停用的 agent
2. 備份相關設定
3. 更新設定為「停用」狀態
4. 記錄停用原因和時間

### Step 4: 生成維護報告

```markdown
# Agent 維護報告 [日期]

## 執行操作
| 操作 | 對象 | 結果 |
|------|------|------|
| [操作1] | [對象] | [結果] |
| [操作2] | [對象] | [結果] |

## 更新後的 Agents 分工
[更新後的職責分配圖或文字描述]

## 建議
- [建議1]
- [建議2]
```

## 常見操作

### 檢視所有 Agents
```
輸出：完整清單 + 分析
```

### 調整 Agent 職責
```
輸入：agent 名稱 + 新職責描述
輸出：更新後的設定 + 驗證結果
```

### 停用 Agent
```
輸入：agent 名稱
輸出：停用確認 + 備份位置 + 更新後清單
```

### 重新命名 Agent
```
輸入：舊名稱 + 新名稱
輸出：更名確認 + 影響範圍 + 更新後設定
```

## 邊界條件

| 情況 | 處理方式 |
|------|----------|
| 找不到指定的 agent | 告知「找不到該 agent」，列出所有可用 agents |
| 修改會影響執行中的任務 | 先確認是否可延後，或在任務完成後再調整 |
| 多個 agents 職責重疊 | 提出整合建議，讓使用者決定 |
| 停用後發現問題 | 提供 rollback 操作，恢復備份設定 |

## 示例

### 典型使用場景

**輸入：**
> 查看一下我目前有哪些 agents

**輸出：**
```markdown
# Agent 清單

| Agent | 職責 | 狀態 |
|-------|------|------|
| Hermes | 深度研究、self-evolution | 啟用 |
| Friday | 日常助理、主要對話 | 啟用 |
| Claude-Code | 程式碼任務 | 啟用 |
```

**建議：**
- Hermes 和 Claude-Code 在研究方面有重疊，建議整合或明確定義分工
