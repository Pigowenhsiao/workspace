# Slide Deck Generator (幻燈片簡報生成器)

## 什麼時候用

- 使用者說「幫我做一個簡報大綱」
- 使用者說「把這篇文章轉成投影片」
- 使用者說「設計視覺提示字」
- 使用者說「生成 Nano Banana Pro 簡報」

## 工作流程

### Step 1: 收集內容

```
必要：
- 主題
- 目標受眾
- 投影片數量（建議 10-30 張）

可選：
- 現有內容檔案或連結
- 品牌規範
- 特定風格要求
```

### Step 2: 定義風格

在撰寫大綱前，必須先定義風格：

```markdown
## 風格指令

你是架構師（The Architect），一個將指令視覺化為高階藍圖風格的精密 AI。

**核心指令：**
1. 分析結構、意圖和關鍵要素
2. 轉化為乾淨、結構化的視覺隱喻
3. 使用克制的調色板和字體
4. 保持 16:9 長寬比
5. 以三聯畫或網格版面呈現

**風格屬性：**
- Design Aesthetic: [描述風格]
- Background: [顏色 #hex]
- Primary Font: [標題字體]
- Secondary Font: [正文字體]
- Color Palette:
  - Primary: [顏色]
  - Accent: [顏色]
- Visual Elements: [視覺元素描述]
```

### Step 3: 產生大綱

```markdown
# [主題] 簡報大綱

## 結構

### Slide 1: 標題頁
- 主標題：[標題]
- 副標題：[副標題]
- 視覺：[背景描述]

### Slide 2: 問題/機會
[內容簡述]

### Slide 3-N: 主體內容
[依序列出]

### Final: 總結/行動號召
```

### Step 4: 視覺提示字

每張投影片提供：

```markdown
## Slide X: [標題]

### 內容區塊
[文字內容]

### 視覺提示字
[具體的視覺設計指示]

### 設計標記
- 配色：[使用的顏色]
- 字體：[字體大小]
- 圖像：[建議的圖像]
```

## 核心原則

| 原則 | 說明 |
|------|------|
| **閱讀優先** | 設計用於「閱讀和分享」，結構應不言自明 |
| **敘事驅動** | 拒絕條列，用故事串聯內容 |
| **完整語境** | 確保視覺能被獨立理解 |
| **中文內容** | 所有內容與佔位符使用中文 |

## 常見結構模板

### 結構 A：問題解決型
```
1. 標題頁
2. 問題定義
3. 現況分析
4. 解決方案
5. 實施路徑
6. 成效預期
7. 總結/行動
```

### 結構 B：時間軸型
```
1. 標題頁
2. 起點/過去
3. 發展歷程
4. 現況
5. 未來規劃
6. 願景
7. 總結
```

### 結構 C：對比型
```
1. 標題頁
2. A 面觀點
3. B 面觀點
4. 整合視角
5. 結論
```

## 邊界條件

| 情況 | 處理方式 |
|------|----------|
| 內容過多 | 濃縮要點，建議拆成多個簡報 |
| 主題不明確 | 先詢問核心訊息 |
| 投影片數量過少 | 建議增加以涵蓋完整敘事 |
| 無品牌規範 | 使用預設專業風格 |

## 示例

**輸出：**
```markdown
# AI 趨勢簡報大綱

## 風格定義

## Slide 1: 標題頁
...

## Slide 2-9: 主體
...

## Slide 10: 總結
```

## 限制說明

- 本 skill 生成大綱和視覺提示字
- 不直接生成圖片（需結合 baoyu 相關 skills）
- 需要 Nano Banana Pro 進行最終渲染
