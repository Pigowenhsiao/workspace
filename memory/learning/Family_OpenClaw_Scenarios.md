# 家庭 OpenClaw 调度场景（家庭管家应用）

来源：社区分享的 OpenClaw 应用场景整理，侧重家庭生活场景的自动化与高效管理。

摘要
- 目标：将 OpenClaw 作为家庭调度中心，统一管理日程、采购、行程、家人动态等信息。
- 价值：减少日程冲突、降低重复工作时间、提高家庭协作效率。

核心场景与实现要点
1) 日程冲突检测
- 场景描述：配偶加班 vs 孩子接送冲突
- 实现要点：在日历/任务间建立冲突检测规则，提供替代方案和提醒。

2) 采购清单管理
- 场景描述：日常消耗监控，达到阈值自动生成购物清单
- 实现要点：结合消费数据与库存管理，定时推送购物清单。

3) 行程协调
- 场景描述：全家出游，规划路线、时间、携带物品清单
- 实现要点：路线计算、时间估算、行李清单自动生成。

4) 家人动态同步
- 场景描述：群聊中的重要信息、行程变更汇总
- 实现要点：聚合要点、推送摘要，方便快速了解全家动态。

5) 生活自动化工作流水线
- 场景描述：将日常重复任务串成流水线
- 实现要点：每天定时检查邮件、日程、周报等，输出初稿或摘要。

6) 个人化健康系统（可选）
- 场景描述：健康数据接入，隐患预警、用药提醒、健康周报
- 实现要点：数据接入、趋势分析、定时提醒。

工作流与实现要点
- 跨场景的共同点：Skill、Hook、MCP 三个组件协同，形成统一入口。
- 安全性与隐私：涉及家庭成员信息，需权限控制与日志审计。
- 演进建议：从日常任务到日常自动化，逐步引入 Heartbeat 与 Approvals。

与 OpenClaw 的对接要点
- Skill 目录结构：将家庭场景拆分成子 Skill，统一触发词和输出模板。
- Hook 与 Heartbeat：用于定期检查、自动执行和状态持久化。
- MCP：对外部服务的接入（如日历、购物服务、健康数据）需明确权限与授权流程。
