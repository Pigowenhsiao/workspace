## ReproRepo：透過 GitHub Issues 實現可擴展的再現性審計
英文標題：ReproRepo: Scaling Reproducibility Audits with GitHub Repository Issues

作者：Shanda Li, Qiuhong Anna Wei, Jingwu Tang, Valerie Chen, Nihar B Shah, Tim Dettmers, Yiming Yang, Ameet Talwalkar
發布日期：2026-06-16
arXiv ID：2606.18237v1
PDF：有

## 英文摘要
Reproducing research results from papers and released code is central to scientific progress. Existing works have introduced benchmarks to evaluate whether LLM agents can assist with reproducibility, but they are difficult to scale due to their reliance on substantial manual effort for data curation and evaluation. We introduce ReproRepo, a scalable framework for reproducibility evaluation that leverages human-raised GitHub issues as naturally occurring supervision on realistic reproduction blockers. We instantiate ReproRepo on 1,149 recent machine learning papers from major conferences and evaluate four frontier model-agent configurations. Our results show that LLM agents, even without executing code, can identify many real-world reproducibility problems from paper-repository pairs: the best agent in our study, namely Codex with GPT-5.5, surfaces at least one semantically related human-reported blocker for ~90% of papers in the study. Further analysis shows that agents are particularly effective for surfacing visible failures and identifying the right semantic region, but may still be insufficient in exact localization. ReproRepo can serve as a reusable, scalable framework for future evaluations of LLM agents on real-world reproducibility auditing.

## 繁體中文摘要
重現論文與發布程式碼的研究成果是科學進展的核心。現有研究已提出基準測試來評估 LLM 代理是否能協助再現性，但由於資料整理和評估需要大量人工努力，這些基準很難擴展。我們提出 ReproRepo，一個可擴展的再現性評估框架，利用人們提出的 GitHub issues 作為對真實再現障礙的自然監督。我們在來自主要會議的 1,149 篇近期機器學習論文中實例化了 ReproRepo，並評估了四種前沿模型-代理配置。我們的結果顯示，即使不執行程式碼，LLM 代理也能從論文-程式碼庫配對中識別出許多真實世界的再現問題：我們研究中最好的代理，即 Codex 搭配 GPT-5.5，能為研究中約 90% 的論文找出至少一個語義相關的人類回報障礙。進一步分析顯示，代理在揭露明顯失敗和識別正確的語義區域方面特別有效，但在精確定位方面可能仍然不足。ReproRepo 可以作為未來對 LLM 代理在真實世界再現性審計方面評估的可重用且可擴展的框架。
